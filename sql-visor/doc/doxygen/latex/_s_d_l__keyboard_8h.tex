{\footnotesize\ttfamily const D\+E\+C\+L\+S\+P\+EC Uint8$\ast$ S\+D\+L\+C\+A\+LL S\+D\+L\+\_\+\+Get\+Keyboard\+State (\begin{DoxyParamCaption}\item[{int $\ast$}]{numkeys }\end{DoxyParamCaption})}

Get a snapshot of the current state of the keyboard.

The pointer returned is a pointer to an internal S\+DL array. It will be valid for the whole lifetime of the application and should not be freed by the caller.

A array element with a value of 1 means that the key is pressed and a value of 0 means that it is not. Indexes into this array are obtained by using S\+D\+L\+\_\+\+Scancode values.

Use \doxyref{S\+D\+L\+\_\+\+Pump\+Events()}{p.}{_s_d_l__events_8h_aa15403d1c8fc341cd9cf750e27a66d19} to update the state array.

This function gives you the current state after all events have been processed, so if a key or button has been pressed and released before you process events, then the pressed state will never show up in the \doxyref{S\+D\+L\+\_\+\+Get\+Keyboard\+State()}{p.}{_s_d_l__keyboard_8h_a69e91052c1f268c9dd47f255e9f9084a} calls.

Note\+: This function doesn\textquotesingle{}t take into account whether shift has been pressed or not.
