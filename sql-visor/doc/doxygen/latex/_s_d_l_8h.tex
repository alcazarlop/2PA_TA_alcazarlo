{\footnotesize\ttfamily D\+E\+C\+L\+S\+P\+EC int S\+D\+L\+C\+A\+LL S\+D\+L\+\_\+\+Init (\begin{DoxyParamCaption}\item[{Uint32}]{flags }\end{DoxyParamCaption})}

Initialize the S\+DL library.

\doxyref{S\+D\+L\+\_\+\+Init()}{p.}{_s_d_l_8h_a8fc8d35348d7c74bad8392d776c937b8} simply forwards to calling \doxyref{S\+D\+L\+\_\+\+Init\+Sub\+System()}{p.}{_s_d_l_8h_adfbfddc0ec609b5e5e5cb1c89298e4db}. Therefore, the two may be used interchangeably. Though for readability of your code \doxyref{S\+D\+L\+\_\+\+Init\+Sub\+System()}{p.}{_s_d_l_8h_adfbfddc0ec609b5e5e5cb1c89298e4db} might be preferred.

The file I/O (for example\+: S\+D\+L\+\_\+\+R\+W\+From\+File) and threading (S\+D\+L\+\_\+\+Create\+Thread) subsystems are initialized by default. Message boxes (S\+D\+L\+\_\+\+Show\+Simple\+Message\+Box) also attempt to work without initializing the video subsystem, in hopes of being useful in showing an error dialog when S\+D\+L\+\_\+\+Init fails. You must specifically initialize other subsystems if you use them in your application.

Logging (such as S\+D\+L\+\_\+\+Log) works without initialization, too.

{\ttfamily flags} may be any of the following OR\textquotesingle{}d together\+:


\begin{DoxyItemize}
\item {\ttfamily S\+D\+L\+\_\+\+I\+N\+I\+T\+\_\+\+T\+I\+M\+ER}\+: timer subsystem
\item {\ttfamily S\+D\+L\+\_\+\+I\+N\+I\+T\+\_\+\+A\+U\+D\+IO}\+: audio subsystem
\item {\ttfamily S\+D\+L\+\_\+\+I\+N\+I\+T\+\_\+\+V\+I\+D\+EO}\+: video subsystem; automatically initializes the events subsystem
\item {\ttfamily S\+D\+L\+\_\+\+I\+N\+I\+T\+\_\+\+J\+O\+Y\+S\+T\+I\+CK}\+: joystick subsystem; automatically initializes the events subsystem
\item {\ttfamily S\+D\+L\+\_\+\+I\+N\+I\+T\+\_\+\+H\+A\+P\+T\+IC}\+: haptic (force feedback) subsystem
\item {\ttfamily S\+D\+L\+\_\+\+I\+N\+I\+T\+\_\+\+G\+A\+M\+E\+C\+O\+N\+T\+R\+O\+L\+L\+ER}\+: controller subsystem; automatically initializes the joystick subsystem
\item {\ttfamily S\+D\+L\+\_\+\+I\+N\+I\+T\+\_\+\+E\+V\+E\+N\+TS}\+: events subsystem
\item {\ttfamily S\+D\+L\+\_\+\+I\+N\+I\+T\+\_\+\+E\+V\+E\+R\+Y\+T\+H\+I\+NG}\+: all of the above subsystems
\item {\ttfamily S\+D\+L\+\_\+\+I\+N\+I\+T\+\_\+\+N\+O\+P\+A\+R\+A\+C\+H\+U\+TE}\+: compatibility; this flag is ignored
\end{DoxyItemize}

Subsystem initialization is ref-\/counted, you must call \doxyref{S\+D\+L\+\_\+\+Quit\+Sub\+System()}{p.}{_s_d_l_8h_a14e9091982e09097985e10e3ca65debb} for each \doxyref{S\+D\+L\+\_\+\+Init\+Sub\+System()}{p.}{_s_d_l_8h_adfbfddc0ec609b5e5e5cb1c89298e4db} to correctly shutdown a subsystem manually (or call \doxyref{S\+D\+L\+\_\+\+Quit()}{p.}{_s_d_l_8h_afdabaf714781099083592051f2d9ac11} to force shutdown). If a subsystem is already loaded then this call will increase the ref-\/count and return.

{\footnotesize\ttfamily D\+E\+C\+L\+S\+P\+EC void S\+D\+L\+C\+A\+LL S\+D\+L\+\_\+\+Quit (\begin{DoxyParamCaption}\item[{void}]{ }\end{DoxyParamCaption})}

Clean up all initialized subsystems.

You should call this function even if you have already shutdown each initialized subsystem with \doxyref{S\+D\+L\+\_\+\+Quit\+Sub\+System()}{p.}{_s_d_l_8h_a14e9091982e09097985e10e3ca65debb}. It is safe to call this function even in the case of errors in initialization.

If you start a subsystem using a call to that subsystem\textquotesingle{}s init function (for example \doxyref{S\+D\+L\+\_\+\+Video\+Init()}{p.}{_s_d_l__video_8h_a05244b30716f0339dcd140c6f92017b7}) instead of \doxyref{S\+D\+L\+\_\+\+Init()}{p.}{_s_d_l_8h_a8fc8d35348d7c74bad8392d776c937b8} or \doxyref{S\+D\+L\+\_\+\+Init\+Sub\+System()}{p.}{_s_d_l_8h_adfbfddc0ec609b5e5e5cb1c89298e4db}, then you must use that subsystem\textquotesingle{}s quit function (\doxyref{S\+D\+L\+\_\+\+Video\+Quit()}{p.}{_s_d_l__video_8h_ade5fc4847320fb235f17265b683fa013}) to shut it down before calling \doxyref{S\+D\+L\+\_\+\+Quit()}{p.}{_s_d_l_8h_afdabaf714781099083592051f2d9ac11}. But generally, you should not be using those functions directly anyhow; use \doxyref{S\+D\+L\+\_\+\+Init()}{p.}{_s_d_l_8h_a8fc8d35348d7c74bad8392d776c937b8} instead.

You can use this function with atexit() to ensure that it is run when your application is shutdown, but it is not wise to do this from a library or other dynamically loaded code.

Shut down specific S\+DL subsystems.

If you start a subsystem using a call to that subsystem\textquotesingle{}s init function (for example \doxyref{S\+D\+L\+\_\+\+Video\+Init()}{p.}{_s_d_l__video_8h_a05244b30716f0339dcd140c6f92017b7}) instead of \doxyref{S\+D\+L\+\_\+\+Init()}{p.}{_s_d_l_8h_a8fc8d35348d7c74bad8392d776c937b8} or \doxyref{S\+D\+L\+\_\+\+Init\+Sub\+System()}{p.}{_s_d_l_8h_adfbfddc0ec609b5e5e5cb1c89298e4db}, \doxyref{S\+D\+L\+\_\+\+Quit\+Sub\+System()}{p.}{_s_d_l_8h_a14e9091982e09097985e10e3ca65debb} and \doxyref{S\+D\+L\+\_\+\+Was\+Init()}{p.}{_s_d_l_8h_a04dc570990c697eed95681216801710d} will not work. You still need to call \doxyref{S\+D\+L\+\_\+\+Quit()}{p.}{_s_d_l_8h_afdabaf714781099083592051f2d9ac11} even if you close all open subsystems with \doxyref{S\+D\+L\+\_\+\+Quit\+Sub\+System()}{p.}{_s_d_l_8h_a14e9091982e09097985e10e3ca65debb}.
